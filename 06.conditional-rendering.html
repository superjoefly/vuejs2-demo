Conditional Rendering

v-if

In Vue, we can render a conditional block using the v-if directive:


      <div id="example1">

        <button @click="toggleSeen">Toggle seen</button>

        <p v-if="seen">Can you see me?</p>

      </div>
      ------------------------------
      var vm = new Vue({
        el: '#example1',
        data: {
          seen: true,
          isSeen: true
        },
        methods: {
          toggleSeen() {
            this.seen = !this.seen;
          },
          toggleIfElse() {
            this.isSeen = !this.isSeen;
          }
        }
      })

Can you see me?

The block will be shown based on the truthiness of the 'seen' data property.

v-else

We can also add an else-block with the v-else directive:


      <div id="example2">

        <button @click="toggleIfElse">Toggle isSeen</button>

        <p v-if="isSeen">First you see me!</p>

        <p v-else>And now you see me!!!</p>

      </div>
      -------------------------------
      var vm = new Vue({
        el: '#example2',
        data: {
          isSeen: true
        },
        methods: {
          toggleIfElse() {
            this.isSeen = !this.isSeen;
          }
        }
      })

First you see me!

And now you see me!!!

The else block must come immediately after the if (or else-if) block.

Conditional Groups Using Template

If we want to toggle more than one element, we can use a template element to wrap the group of elements to be toggled. The final rendered result will not include the template element:


      <div id="example3">

        <button @click="toggleSeen">Toggle</button>

        <template v-if="seen">

          <h3>Heading</h3>

          <p>Paragraph 1</p>

          <p>Paragraph 2</p>

        </template>

        <p v-else>This is rendered using v-else!</p>

      </div>
      ------------------------------
      var vm = new Vue({
        el: '#example3',
        data: {
          seen: true
        },
        methods: {
          toggleSeen() {
            this.seen = !this.seen;
          }
        }
      })

This is rendered using v-else!

We can still use v-else with the template element.

v-else-if

The v-else-if directive serves as an else-if block for the v-if directive. It can be chained multiple times:

Note: this example also uses v-model:


      <div id="example4">

        <p>Type <b>A, B</b> or <b>C</b> into the input field!</p>

        <input type="text" v-model="type" />

  

        <p v-if="type === 'A' || type === 'a'">Type is A!</p>

        

        <p v-else-if="type === 'B' || type === 'b'">Type is B!</p>

        <p v-else-if="type === 'C' || type === 'c'">Type is C!</p>

  

        <p v-else>Type is not A, B or C!</p>

      </div>
      -----------------------------
      var vm = new Vue({
        el: '#example4',
        data: {
          type: ''
        }
      })

Type A, B or C into the input field!

Type is A!

Type is B!

Type is C!

Type is not A, B or C!

The v-else-if block must immediately follow a v-if block or another v-else-if block.

Using key to Control Reusable Elements

To improve performance, Vue tries to render elements as efficently as possible, often re-using them instead of creating them from scratch. This is often useful, however it is not always desirable. Using a key attribute with unique values will inform Vue that the elements are completely separate and that they should not be re-used:


      <div id="example5">

        <template v-if="loginType === 'username'">

          <label>Username</label>

          <input placeholder="Enter your username..." key="username-input" />

        </template>

        

        <template v-else>

          <label>Password</label>

          <input type="password" placeholder="Enter your password..." key="password-input" />

        </template>

        <br />

        <button @click="toggleInput">Toggle</button>

      </div>
      -------------------------------
      var vm = new Vue({
        el: '#example5',
        data: {
          loginType: 'username'
        },
        methods: {
          toggleInput() {
            if (this.loginType === 'username') {
              this.loginType = 'password';
            } else {
              this.loginType = 'username';
            }
          }
        }
      })

The inputs above will be rendered from scratch each time you toggle.

Note: the label elements are still efficiently re-used because they don't have key attributes.

v-show

Another Vue.js directive that can be used for conditionally displaying an element is v-show:


      <div id="example6">

        <button @click="toggleShown">Toggle</button>

        <p v-show="isShown">I use <b>v-show</b>!</p>

      </div>
      ------------------------------
      var vm = new Vue({
        el: '#example6',
        data: {
          isShown: true
        },
        methods: {
          toggleShown() {
            this.isShown = !this.isShown;
          }
        }
      })

I use v-show!

The difference is that an element using v-show will always be rendered and remain in the DOM...v-show simply toggles the CSS display property of the element.

Note: the v-show directive does NOT support the template element syntax, nor does it work with v-else.

More Notes:

  • v-if is said to be lazy because the element it is attached to will not be rendered unless its bound condition becomes truthy. This is good if you don't need something to be rendered immeditately, but it's expensive if you need to toggle something often.
  • Using v-show can be more expensive at runtime, but it is a better option if something is going to be toggled often.
  • When using v-for together with v-if, the v-for directive has a higher priority. This will be discussed more in the next chapter.
